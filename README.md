# e57 → Nav2 Static Costmap (ProjectRED edition)

Convert an `.e57` point cloud into a **Nav2 static costmap** (`map.png` + `map.yaml`) where grayscale encodes **slope**:

* `0`   = flat (free)
* `254` = vertical (lethal)
* `255` = unknown

`map.yaml` uses `mode: raw` so the grayscale values are consumed directly by `map_server`.

New features:

* **Free/occupied slope thresholds** — map slope to cost values based on rover climbability.
* **Automatic trinary YAML output** (`map_trinary.yaml`) with configurable `free_thresh` / `occupied_thresh` for Nav2’s `map_server` in `mode: trinary`.
* **Landmark-based overrides** — optionally mark specific XY coordinates as *definitely obstacles* or *definitely free* within a given radius.

## Configure

Edit **CONFIG CONSTANTS** in `e57_to_nav2_costmap.py`:

* `INPUT_E57` — input E57 path
* `RESOLUTION_M` — grid resolution (m/px)
* `MIN_SAMPLES_CELL` — min points per cell to be valid
* `GAUSS_SIGMA_CELLS` — Gaussian blur sigma (cells)
* `CELL_REDUCER` — `median|min|mean`
* `FLIP_Y` — mirror across XZ if Y inverted
* `CROP_BOUNDS` — optional crop

**New slope-to-cost options:**

* `FREE_SLOPE_DEG` — fully free threshold
* `OCC_SLOPE_DEG` — lethal threshold
* `GAMMA_RAMP` — ramp curve factor

**New trinary YAML options:**

* `TRINARY_FREE_THRESH` — fraction ≤ this is free in trinary mode
* `TRINARY_OCCUPIED_THRESH` — fraction ≥ this is occupied in trinary mode

**Landmark override options:**

* `USE_LANDMARK_OBSTACLES` — `True/False` to enable obstacle marking
* `USE_LANDMARK_FREE` — `True/False` to enable free marking
* `LANDMARK_OBSTACLES` — list of `(x, y)` coordinates (m) to mark as obstacle
* `LANDMARK_FREE` — list of `(x, y)` coordinates (m) to mark as free
* `LANDMARK_RADIUS_M` — radius (m) around each landmark to apply override

## Sure overlays (landmarks, starts, waypoints)

You can force cells to be **obstacles** or **free** based on landmark annotations.

**How it works**

* Provide an **annotation PNG** with colored dots at world XY positions.
* The script converts those dots into masks and applies them on top of the computed costmap.
* Radii are given in meters and converted to pixels using `RESOLUTION_M`.

**Assumptions**

* The annotation image is aligned to the generated grid (same width/height as `map.png`) **or** it is generated by the script’s helper (recommended).

**Config constants (new):**

* `USE_SURE_OVERLAYS` — enable/disable this feature (`True|False`).
* `ANNOTATION_PNG` — path to the annotation image (e.g., `~/Desktop/projectred/map_converter_pj/data/landmarks.png`).
* `SURE_OBS_RADIUS_M` — radius (m) around **landmark** points to force **obstacle** (cost `254`).
* `SURE_FREE_RADIUS_M` — radius (m) around **start/waypoint** points to force **free** (cost `0`).
* `ANNO_COLORS` — RGB tuples used to detect classes:

  * `"obstacle"`: `(255, 0, 0)`  (red)
  * `"start"`:    `(0, 0, 255)`  (blue) → treated as **free**
  * `"waypoint"`: `(0, 255, 0)`  (green) → treated as **free**

**Priority rules**

* Free overrides apply first, then obstacle overrides. (Change order in code if you prefer.)
* Unknown cells can be forced to free/obstacle by overlays.

**Generate a blank aligned overlay (optional helper)**
The script can emit a blank `overlay_template.png` sized like the costmap so you can paint points in any editor. Enable the helper flag in the script, run once, then draw markers with the colors above.

## Notes

* Unknowns: `< MIN_SAMPLES_CELL` → 255 (unknown)
* Masked Gaussian smoothing avoids bleeding across unknowns
* Slope mapping: free ≤ `FREE_SLOPE_DEG`, lethal ≥ `OCC_SLOPE_DEG`, ramp in between
* Trinary mode lets Nav2 threshold grayscale into free/occupied
* Unknowns: `< MIN_SAMPLES_CELL` → 255 (unknown)
* Masked Gaussian smoothing avoids bleeding across unknowns
* Slope mapping: free ≤ `FREE_SLOPE_DEG`, lethal ≥ `OCC_SLOPE_DEG`, ramp in between
* Trinary mode lets Nav2 threshold grayscale into free/occupied
* Landmark overrides allow enforcing known free/obstacle areas regardless of slope data

## Troubleshooting

* `No module named pdal` → activate env
* Noisy map → increase `RESOLUTION_M` or lower `MIN_SAMPLES_CELL`
* E57 read errors → ensure `libe57format` + `xerces-c` installed or convert to LAZ
